# Backlog: Notifications

## Priority
Low — planned enhancement

## Description
Notify the person who owns the next action when a PR (or any document) moves to their stage.

**Example:** When a PR is submitted and waiting for manager approval, the manager receives a notification. When returned to the requester, the requester is notified.

## Scope
- PR approval stages (all steps)
- PO approval stages
- Billing review and confirmation steps

## Channel
TBD — email / in-app notification / both

## Notes
- Notification should include: PR/PO number, topic, amount, current stage, and a direct link
- Do not notify people who are not the current action owner
